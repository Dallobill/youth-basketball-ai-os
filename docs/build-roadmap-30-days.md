# 30-Day Build Roadmap

## Week 1 — Data foundation
Goal: get structure right.

Build:
- organizations
- seasons
- teams
- players
- coaches
- player-team assignment
- evaluations table
- practices table
- games table
- attendance tables

Success criteria:
- you can add a player
- log a practice
- submit an evaluation
- view saved records

## Week 2 — Coaching workflow
Goal: make the workflow useful.

Build:
- practice notes
- game notes
- clip records
- player tags
- team tags
- summary endpoints

Success criteria:
- one coach can enter post-game notes in under 5 minutes
- one player has a full development record timeline

## Week 3 — AI outputs
Goal: generate useful language.

Build:
- player summary prompt
- team summary prompt
- practice plan prompt
- parent update prompt

Success criteria:
- summaries are specific, short, and actionable
- outputs sound like a real coach, not generic AI fluff

## Week 4 — Pilot + feedback
Goal: test in real use.

Build:
- simple dashboard or admin page
- report export endpoint
- improvement feedback loop

Success criteria:
- one full team uses it for 2–4 weeks
- at least 5 summaries are reviewed by a coach
- you refine the prompt based on real usage
